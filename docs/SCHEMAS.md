## JSON-схемы платформы

### FactItem

- **id**: целое, опционально (БД-идентификатор).
- **study_id**: целое, идентификатор исследования.
- **key**: строка, ключ факта (например, `primary_endpoint`).
- **value**: произвольный JSON-объект со структурой факта.
- **anchors**: массив ссылок на якоря:
  - `anchor_id`: строка, см. правило формирования anchor_id.

### SoAResult

- **visits**: массив объектов:
  - `visit_id`: строка.
  - `name`: строка.
  - `day`: целое или null.
- **procedures**: массив объектов:
  - `code`: строка.
  - `name`: строка.
- **matrix**: массив ячеек:
  - `visit_id`: строка (ссылка на visits).
  - `procedure_code`: строка (ссылка на procedures).
  - `value`: `"X" | "O" | "-" | null`.
  - `anchor_ids`: массив строк-якорей.

### Artifacts

- **claims**: массив объектов:
  - `id`: строка.
  - `text`: строка (claim).
- **citations**: массив `anchor_id` (строки).
- **numbers**: массив чисел (извлечённые числовые значения).

### QCReport

- **items**: массив объектов:
  - `code`: строка (идентификатор правила QC).
  - `message`: строка (человеческое описание).
  - `severity`: `"info" | "warning" | "error"`.
  - `anchor_ids`: массив строк-якорей, к которым относится замечание.
- **is_blocking**: boolean, можно ли публиковать результат без исправления.
